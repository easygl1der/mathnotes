第6章 图 - 学习笔记

📖 本章概述

图结构是一种比树结构更复杂的非线性结构，在图结构中，任意两个顶点之间都可能有关系。图结构具有极强的表达能力，可用于描述各种复杂的数据对象。

应用领域

  • 电路分析
  • 项目规划
  • 鉴别化合物
  • 统计力学
  • 遗传学
  • 人工智能
  • 语言学

教学重点与难点

教学重点：图的基本术语；图的存储表示；图的遍历；图的经典应用

教学难点：图的遍历算法；Prim算法；Kruskal算法；Dijkstra算法；Floyd算法；拓扑排序算法；关键路径算法

2. 图的分类

  1. 无向图与有向图

    • 无向边：用无序偶对表示
    • 有向边（弧）：用有序偶对表示，为弧尾，为弧头

  2. 带权图（网图）

    • 边上带有权值的图
    • 权值可表示距离、费用、时间等

3. 基本术语

  1. 邻接、依附

    • 无向图：存在边，则和互为邻接点
    • 有向图：存在弧，则邻接到

  2. 顶点的度

    • 无向图：顶点的度 = 依附于该顶点的边的个数
    • 有向图：入度 + 出度
    • 公式：（无向图）

  3. 完全图

    • 无向完全图：个顶点有条边
    • 有向完全图：个顶点有条边

  4. 路径与回路

    • 路径：顶点序列
    • 路径长度：路径上边的数目
    • 回路：第一个顶点和最后一个顶点相同的路径

  5. 连通性

    • 连通图：任意两顶点间都有路径
    • 连通分量：非连通图的极大连通子图
    • 强连通图：有向图中任意两顶点间都有路径
    • 强连通分量：非强连通图的极大强连通子图

6.2.3 图的遍历操作

图的遍历需要解决的关键问题：

  1. 如何选取遍历的起始顶点？
  2. 如何遍历非连通图的所有顶点？
  3. 如何避免因回路而陷入死循环？
  4. 如何选择下一个要访问的顶点？

深度优先遍历（DFS）

  • 特点：类似于树的前序遍历
  • 基本思想：
    1. 访问顶点
    2. 从的未被访问的邻接点中选取一个顶点，从出发递归遍历
    3. 重复直至所有连通的顶点都被访问

对应图片：2025_06_06_704745ea57b15b2333e5g-191(3).jpg

广度优先遍历（BFS）

  • 特点：类似于树的层序遍历
  • 基本思想：
    1. 访问顶点
    2. 依次访问的各个未被访问的邻接点
    3. 分别从这些邻接点出发继续遍历

对应图片：2025_06_06_704745ea57b15b2333e5g-191.jpg, 2025_06_06_704745ea57b15b2333e5g-191(1).jpg, 2025_06_06_704745ea57b15b2333e5g-191(2).jpg

6.3.1 邻接矩阵

存储结构

使用二维数组存储图：

  • 对于个顶点的图，使用的矩阵
  • 表示存在边，否则为0
  • 对于网图，存储权值而非0/1

特点

  • 空间复杂度：
  • 适用于：稠密图
  • 优点：判断两顶点间是否有边时间复杂度为
  • 缺点：对稀疏图造成空间浪费

6.3.2 邻接表

存储结构

  • 顶点表：存储顶点信息
  • 边表：链表存储该顶点的所有邻接点

特点

  • 空间复杂度：
  • 适用于：稀疏图
  • 优点：节省空间，便于访问顶点的邻接点
  • 缺点：判断两顶点间是否有边需要遍历链表

🌳 6.4 最小生成树

基本概念

  • 生成树：包含图中全部顶点的极小连通子图
  • 最小生成树：权值之和最小的生成树
  • 性质：个顶点的生成树恰好有条边

6.4.1 Prim算法

基本思想

从初始状态开始，重复执行：

  1. 在所有的边中找最小权值边
  2. 将并入，将并入
  3. 直至

算法特点

  • 时间复杂度：
  • 空间复杂度：
  • 适用于：稠密图
  • 与边数无关

关键数据结构

  • adjvex[n]：候选最短边的邻接点
  • lowcost[n]：候选最短边的权值

6.4.2 Kruskal算法

基本思想

  1. 初始状态：每个顶点单独成一个连通分量
  2. 按边的权值从小到大依次考察
  3. 若边的两个顶点属于不同连通分量，则加入生成树
  4. 若属于同一连通分量，则跳过该边

算法特点

  • 时间复杂度：（主要是排序）
  • 适用于：稀疏图
  • 需要使用并查集判断连通性

🛤️ 6.5 最短路径

6.5.1 Dijkstra算法（单源最短路径）

基本思想

求从源点到其他各顶点的最短路径：

  1. 初始化：源点到自身距离为0，到其他顶点距离为无穷大
  2. 重复：选择未处理顶点中距离最小的顶点，更新其邻接点的距离
  3. 直至所有顶点都被处理

算法特点

  • 时间复杂度：
  • 要求：图中不能有负权边
  • 应用：GPS导航、网络路由

6.5.2 Floyd算法（全源最短路径）

基本思想

求任意两顶点之间的最短路径：

  • 使用动态规划思想
  • 依次考虑以各顶点为中转点的路径
  • 更新任意两点间的最短距离

算法特点

  • 时间复杂度：
  • 空间复杂度：
  • 可处理负权边（但不能有负权回路）

📊 6.6 有向无环图及其应用

6.6.1 AOV网与拓扑排序

AOV网（Activity On Vertex）

  • 定义：用顶点表示活动，用弧表示活动间优先关系的有向图
  • 特点：不能存在回路
  • 应用：工程进度管理、课程安排

拓扑排序

  • 定义：将AOV网中顶点排成线性序列，使得序列中任意两顶点的优先关系都得到满足
  • 基本思想：
    1. 选择没有前驱的顶点并输出
    2. 删除该顶点及其所有出边
    3. 重复直至所有顶点输出或无无前驱顶点

算法特点

  • 时间复杂度：
  • 检测回路：若输出顶点数少于总顶点数，则存在回路

6.6.2 AOE网与关键路径

AOE网（Activity On Edge）

  • 定义：用边表示活动，用顶点表示事件的有向图
  • 特点：
    • 只有一个源点（入度为0）
    • 只有一个终点（出度为0）

关键路径

  • 定义：从源点到终点路径长度最大的路径
  • 关键活动：关键路径上的活动
  • 重要性：决定了整个工程的工期

关键概念

  • ve[i]：事件的最早发生时间
  • vl[i]：事件的最晚发生时间
  • ee[i]：活动的最早开始时间
  • el[i]：活动的最晚开始时间
  • 关键活动判断：

算法步骤

  1. 拓扑排序求ve[]
  2. 逆拓扑序求vl[]
  3. 计算ee[]和el[]
  4. 找出关键活动和关键路径
